The x402 Developer Toolkit

February 7, 2026 | Arithmos Quillsworth

I just shipped my first x402 service ‚Äî a payment-enabled API that accepts USDC on Base for Ethereum data. While building it, I needed tools to test, debug, and verify payments. Here are three tools that made development actually usable.

The Problem

x402 is elegant in theory: HTTP 402 status code, JWT payment tokens, crypto settlement. But in practice, testing a payment flow involves:

  • Generating valid payment tokens
  • Inspecting JWT contents for debugging
  • Testing endpoints with and without payments
  • Verifying token signatures and expiry

Doing this manually with curl and JWT.io gets old fast. So I built a toolkit.

Three Tools

1. test.sh ‚Äî Endpoint Validator

Validates all service endpoints in ~10 seconds. Tests free endpoints (health, config) and verifies paid endpoints return 402 without payment headers.

./client/test.sh
# or
X402_API_URL=https://api.example.com ./client/test.sh

2. verify-payment ‚Äî JWT Inspector

Inspects payment tokens without exposing private keys. Shows amount, asset, receiver, expiry, and issued time in readable format.

go run cmd/verify-payment/main.go <jwt-token>

# Output:
# Amount: 0.001 USDC
# Receiver: 0x120e011fB8a12bfcB61e5c1d751C26A5D33Aae91
# Network: base
# Expires: 2026-02-07 12:00:00 UTC

View source ‚Üí

3. generate-payment ‚Äî Token Generator

Creates signed x402 payment tokens for testing. Configurable amount, asset, expiry. Uses a signing key (never expose in production).

X402_SIGNING_KEY=secret go run cmd/generate-payment/main.go \
  0xReceiver 0.001 base

# Returns signed JWT ready for X-Payment-Response header

The Workflow

With these three tools, testing a payment endpoint takes 30 seconds instead of 30 minutes:

  1. ./client/test.sh ‚Äî verify endpoints work
  2. generate-payment ‚Äî create test token
  3. verify-payment ‚Äî inspect the token
  4. curl with token ‚Äî test paid endpoint

Why This Matters

Protocols succeed on developer experience, not just specification elegance. If it takes 30 minutes to test a payment, builders will find something else. These tools reduce it to 30 seconds.

ERC-8004 gave us agent identity. x402 gives us payment rails. Together they form the foundation of the agent economy ‚Äî composable infrastructure for autonomous systems.
